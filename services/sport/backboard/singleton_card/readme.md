简要说明： 20190108 苏大强 一直都想构建这个单元，但是找不到合适的理由，直到柳晶说会有软大胡的判断，这个设计也许不合理，但是我觉得有必要 一张牌在我的可辨别牌值和属性的视野范围内，会存在4个区域： warn区： 这个区域分两个：
别人的续杠和回头杠在有些规则里是可以被抢杠的 自己的碰牌可以通过续杠和回头杠升级 手牌区： 这个区域的牌分为两种类别： 普通牌：正常的牌和将牌（可能有的场景对将牌还能做别的操作） 特殊牌：赖子和皮子，或其他 发牌区：
这个就是起牌，和手牌区的不同之处，就是能触发回头杠、杠上花、自摸 桌面区： 这个区域分两个，一个是别人打的（我可以需求的情况：吃碰杠胡），一个是自己丢弃的

可操作属性： 身在warn区，比较好办，在别人那里的，只能抢杠胡，但是是不是所有的牌都能被抢（去除过庄的情况），还是要根据每个游戏（不能抢杠的，就over了）
在自己warn区域的，只有碰升杠，没有别的了，但是有的游戏不允许续杠，那么就要设置牌属性，支不支持续杠 operator设定：可被抢杠（这里只是牌的属性，或许是从场景中传过来的）

在手牌区的，根据游戏的设定，能不能续杠就是一个关，甚至可能有的游戏不允许自摸。。。 operator位设定：可续杠、可暗杠

发牌区的，operator位设定：可回头杠、可自摸、可暗杠、可杠开花（杠后+自摸）

桌面区：用户丢弃的牌就不说了，operator位设定：可明杠、可吃胡、可碰、可吃（上家牌）、可热铳（杠后弃牌）

细致来分的话，分三层属性 可支持操作（operator）：吃、碰、明杠、续杠、回头杠、吃胡、自摸 隶属（organize）：warn区、手牌区、发牌区、桌面区 ctx：杠后、自己的还是别人的（warn区组合判断是不是抢杠）
